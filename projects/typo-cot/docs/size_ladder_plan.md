# サイズ梯子拡張計画: モデルサイズを脅威から検証へ(P1〜P6)

本書は、ユーザーが 2026-07-19 に提示した**サイズ梯子拡張方針**の正典文書である。Phase A/B
(実験1〜18、`docs/experiment_plan.md` / `docs/experiments_11_18_plan.md`)は 1B〜8B のモデルで
統一仮説 **ERDC連鎖(Encode–Repair–Divert–Carry)+ 3モデレーター M1/M2/M3** を確立した。
本計画は、**同一の機構が容量スケールに対しても成立するか**を、サイズ梯子(Gemma-3 1B→27B 主軸)で
確証的に試験する。核心は発想の転換にある — 「モデルサイズ」は本研究**最大の外的妥当性の穴**だが、
統一仮説が正しければサイズ効果は3パラメータを通じてのみ現れるはず(P6)。ゆえにサイズ梯子は
反論封じでなく、仮説の**予測力を賭ける確証的追試**になる。

- 実装先: `projects/typo-cot`(モジュール `typo_cot`)
- 事前登録: `docs/hypothesis_registry.md` §Size-Ladder Extension(**P1〜P6** + P1 反証受け皿)
- 連鎖定義・実験対応: `docs/experiments_11_18_plan.md`(ERDC / M1・M2・M3 / 実験11〜18)

### 対応表(P ↔ 実験 ↔ 予測骨子 ↔ データソース)

| P | 短名 | 実験(@スケール) | 予測骨子 | 主データソース |
|---|---|---|---|---|
| **P1** | 背骨の防衛線 | 実験1 @27B×MATH | フロンティア近傍で **IE/TE ≥ 0.7** 保持 | 実験1 IE/TE/DE(27B×MATH) |
| **P2** | ショートカットのサイズ則 | 実験14 + 実験1 @梯子(MMLU) | **DE/TE がサイズ単調増加**、M3統制で size係数消失 | 実験1 DE/TE + 実験14 no-CoT |

CoTが負荷を担う条件(タスク難度が能力に拮抗する領域)で成立し、境界は M3 で予測される」とスコープ
(Lanham整合の境界条件発見)。詳細は `docs/hypothesis_registry.md` §Falsification Receptacle。

## 0. なぜサイズが最大の穴か

Phase A の全モデルは 1B〜8B に留まる。査読者は、とりわけ **Lanham et al.** の知見(容易タスクでは
faithfulness がモデルサイズと共に低下する)を引いて、「本研究の見出しである **IE支配**は小型モデルの
アーティファクトで、大型モデルでは崩れるのではないか」と問うだろう。**モデルサイズは未探索の軸として
最大の外的妥当性の穴**であり、本計画はこれを正面から検証に変える。

### 0.1 実験別サイズ感度表

| 実験 | サイズ感度 | 何が動きうるか / 理由 |
|---|---|---|
| 実験1(IE/TE/DE 媒介) | **高** | IE支配が見出し。Lanham効果で大型ほど DE(ショートカット)が増える可能性。**P1・P2 の主戦場** |
| 実験4(fixed-target Δρ) | **高** | family×format 交互作用(H4)がサイズで変質しうる。MC の選択肢汚染が容量で変わる |
| 実験2(削除RD) | 中 | 読み出し集中度がサイズで下がれば削除RDも下がる(P5)。**方向は予測可能** |
| 実験8(早期層局在) | 中〜低 | 局在は頑健と期待(P3)。層数増でも相対深さ<35% を予測。**低リスク** |
| 実験14(no-CoTショートカット) | **高だが検証機会** | DE/TE のサイズ則(P2)を直接説明する軸。脅威であると同時に**最大の検証機会** |
| 実験7(校正ボトルネック R_Q偏在) | 低 | 補正器の挙動はモデルサイズにほぼ非依存。梯子の優先度低 |
| 実験12(R_C組成 M2) | 中 | 結論句/数値シェアがサイズで動きうる。LOO版で追試 |

### 0.2 既観測のサイズ勾配

- **MC の DE/TE**: 1B系 **0.4〜0.6** → 3〜7B **0.2〜0.4**(観測レンジでは**低下**)。
- **注(レジーム問題)**: この観測方向(1B→7B で低下)と P2 の予測方向(**サイズ単調増加**)は一見
  逆である。これがまさに梯子で解くべき問いであり、小型域(1B→7B)での低下が、大型域(12B→27B)で
  **Lanham レジーム**に入って反転するかを P2 は問う(P2 は MMLU 特化での大型域反転を予測)。

### 0.3 Lanham リスク

- **Lanham et al.**: 容易タスクでは faithfulness(答えの CoT 依存)がモデルサイズと共に低下する —
  大型モデルは CoT なしでも解けるため、CoT が答えを支配しなくなる。
- **見出しへの脅威**: 「IE支配は小型のアーティファクトで、27B では崩れる」というレビュー筋。
- **転化**: この脅威を **P1(フロンティア近傍で IE 保持)**+**P2(容易 MC でのショートカット増を M3 で
  予測・吸収)**で正面から検証に変える。faithfulness 低下は"予測される現象"として枠組みに取り込む。

## 1. 梯子設計

### 1.1 梯子点

- **主軸**: Gemma-3 **1B → 4B → 12B → 27B**(4点。同一系統で容量だけ変える最良の統制)。
- **予備**: Qwen2.5 **7B → 14B → 32B**(3点。系統間再現=family 一般化)。
- **スポット**: Llama-3.3-**70B int4**(最大点の1発確認。int4 量子化でメモリ節約)。

| 系統 | 点 | 概算層数 | 役割 |
|---|---|---|---|
| Gemma-3 | 1B | 26 | Phase A 基点(既取得) |
| Gemma-3 | 4B | 34 | Phase A 基点(既取得) |
| Gemma-3 | 12B | 48 | 第1波・**backward 可の最大点** |
| Gemma-3 | 27B | 62 | 第1波・**主目標**(P1/P3/P6) |
| Qwen2.5 | 7B / 14B / 32B | — | 第2波・系統間再現 |
| Llama-3.3 | 70B(int4) | 80 | スポット確認 |

### 1.2 ベンチマーク

- **{MATH, MMLU} を主軸**、**GSM8K 併走**。
  - **MATH** = 27B の**能力フロンティア近傍**(P1 の主戦場。CoT が実際に仕事をする場所)。
  - **MMLU** = 容易 MC(P2 のショートカット則・Lanham レジームの主戦場)。
  - **GSM8K** = 併走(Phase A との連続性。ただし天井に注意、下記 1.3)。

### 1.3 天井効果とエスカレーション

- 27B は **GSM8K でクリーン精度 ≈ 95%** → typo で flip する母数が小さく検定力不足。
- **ルール**: ある設定で **flip 件数 < 50** になったら **LXT-8 へエスカレーション**(4語→8語摂動で
  摂動強度を上げ、flip 母数を確保)。MATH/MMLU でも同ルールを適用。

## 2. forward-only 化表

大型点(27B/32B)では **backward(勾配帰属)がメモリ的に高価**。原則 **forward-only 化**する。
AttnLRP R_C が必要な軸は **LOO(leave-one-out、forward のみのアブレーション重要度)へ差し替える**。

| 実験 / 軸 | 27B・32B 実行法 | 備考 |
|---|---|---|
| 実験1(媒介) | **そのまま** | CoT 移植は forward のみ |
| 実験3(KL) | **そのまま** | KL は forward のみ |
| 実験9(修復) | **そのまま** | 隠れ状態 cos は forward のみ(Gemma天井のため `repair_speed_95_rel` を使用) |
| 実験14(no-CoT) | **そのまま** | 答えスパン強制は forward のみ |
| 実験8(patching) | **そのまま** | activation patching は forward のみ |
| 実験2(削除) | **LOO 差し替え** | top-R_C 選定を LOO 重要度で(AttnLRP 不要) |
| 実験12(R_C組成) | **LOO 差し替え** | R_C を LOO 版で組成分類 |
| 観察軸(R_C分布 全般) | **LOO 差し替え** | 27B で AttnLRP を回さない |
| 実験4(fixed-target Δρ) | **LOO版 Jaccard@10 代替** | fixed-target R_C を LOO版 Jaccard@10 で近似。H6 で LOO が実験4の ρ(J\|R) 構造を**全設定再現済み**=代替の正当化 |

### 2.1 R_Q フォールバック連鎖

R_Q(typo 語の帰属)は backward を要するため、大型点では次の連鎖で退避する:

1. **12B backward 可** — 12B までは通常の backward で R_Q を取得。
2. **27B: gradient checkpointing 試行** — メモリ削減で backward を通す試み。
3. 通らなければ **G×I(Gradient×Input)** — より軽量な勾配帰属へ後退。
4. それも不可なら **transfer-LXT + Random-4** — 小型点で得た LXT 標的を 27B に転移し、対照は
   Random-4。R_Q 実測を諦め、転移標的で代替する。

## 4. 設計思想: サイズを脅威から検証へ転化する

- **発想**: 「モデルサイズ」を守勢に回る穴でなく、統一仮説の**予測力の試験台**に変える。ERDC+M1/M2/M3
  が正しければ、サイズ効果は3パラメータを通じてのみ現れる(**P6**)。よってサイズ梯子は反論封じでなく
  **確証的追試**である。
- P1〜P6 は事前登録済みの **falsifiable predictions**(`docs/hypothesis_registry.md`)。各々に反証分岐、
  **P1 には反証受け皿**(レジームスコープ=Lanham 境界条件の発見)を凍結済み。

### 4.1 確証的追試レイヤーの統計分離

梯子分の解析は主分析(Phase A/B)と**層を分ける**。理由は2つ:

1. **Holm/GLMM 再計算連鎖の回避**: 梯子点を主分析のプールに混ぜると、Holm 族の族横断補正と GLMM の
   ランダム効果構造を**全て再計算する連鎖**が発生する。梯子を別レイヤーに保つことでこれを回避する。
2. **後出し批判の回避**: P1〜P6 を事前登録し梯子を独立レイヤーに保つことで、「サイズを見てからゴールを
   動かした」という**後出し批判を構造的に防ぐ**。主分析の判定(H1〜H18)は梯子結果に依存せず不変。

- **位置づけ**: 梯子は **confirmatory replication layer**。主分析を1ミリも書き換えず、その上に
  「同じ機構がスケールしても成り立つ」ことを重ねる。

## 5. フォールバック(梯子が間に合わない場合)

締切までに梯子が回りきらない場合の事前合意:

1. **スコープ明示** — 本文の主張を検証済みのサイズ域(1B〜8B、+回った梯子点まで)に明示的に限定する。
2. **P1〜P6 を falsifiable predictions として本文明記** — 未実施でも、6予測を「本枠組みが生む反証可能な
   サイズ予測」として本文(Discussion / Predictions)に記載し、査読者・後続研究が検証できる形で残す。
3. **Limitations で Lanham リスクを自己指摘** — 「容易タスクで faithfulness がサイズと共に低下しうる
   (Lanham et al.)。我々の IE 支配が大型域で P2 の予測通りショートカットへ移行する可能性は未検証で、
   これは本研究の限界」と自己申告する。守勢でなく、**予測と限界を明示することで誠実性を担保**する。
